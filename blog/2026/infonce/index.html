交叉熵（Cross Entropy）、KL 散度（KL Divergence） 和 熵（Entropy）

简单来说，交叉熵（Cross Entropy）、KL 散度（KL Divergence） 和 熵（Entropy） 之间有一个非常核心的关系公式： \(\text{交叉熵} = \text{熵} + \text{KL 散度}\) \(H(P, Q) = H(P) + D_{KL}(P \mid \mid Q)\) 其中 \(P\) 通常代表真实分布（标签），\(Q\) 代表模型预测的分布。

1. 核心区别

我们可以从“信息论”的角度来直观理解：

概念 直观定义 物理含义
熵 \(H(P)\) 表达真实分布 \(P\) 所需的最小平均信息量。 事情本身的确定性/混乱度。
交叉熵 \(H(P, Q)\) 用预测分布 \(Q\) 的策略去对真实分布 \(P\) 的信息进行编码，所需的实际平均信息量。 模型预测的代价。
KL 散度 \(D_{KL}(P \\mid Q\) 因为预测分布 \(Q\) 不够准，导致比最小信息量多出来的部分。 两个分布之间的“距离”或“差异”。

主要差异点：

  • 物理意义：KL 散度衡量的是两个概率分布之间的差异；交叉熵衡量的是在已知 \(P\) 的情况下，用 \(Q\) 来表示 \(P\) 的平均成本。
  • 数值范围：KL 散度的最小值是 0；交叉熵的最小值是真实分布本身的熵 \(H(P)\)。
  • 对称性：两者都不具备对称性，即 \(D_{KL}(P \mid Q) \neq D_{KL}(Q \mid P)\)。

2. 什么时候它们会相等？

在数学公式上，交叉熵和 KL 散度永远不会严格数值相等（除非 \(H(P) = 0\)，即真实分布是完全确定的，比如 One-hot 编码）。 但在机器学习优化（Loss Function）的场景下，我们常说“最小化交叉熵等价于最小化 KL 散度”，这是因为：

  • 在训练模型时，\(P\) 是真实标签的分布，它是固定的（常量）。
  • 既然 \(H(P)\) 是个常数，那么在求导优化时，\(\nabla H(P, Q) = \nabla D_{KL}(P \mid \mid Q)\)。
  • 因此，从优化目标（梯度的方向）来看，它们是完全相等的。

3. 为什么分类任务常用交叉熵而不是 KL 散度？

虽然优化效果一样，但代码实现中通常首选交叉熵：

  1. 计算更简单：交叉熵公式比 KL 散度少了一项 \(\sum P(x)\log P(x)\)，计算量更小。
  2. 数值更直观：在分类任务中，标签通常是 One-hot 向量（例如 \([1, 0, 0]\)），此时 \(H(P) = 0\)。在这种特殊情况下，交叉熵的数值恰好等于 KL 散度的数值。

总结公式对比

  • 熵：\(H(P) = -\sum P(x) \log P(x)\)
  • 交叉熵：\(H(P, Q) = -\sum P(x) \log Q(x)\)
  • KL 散度：\(D_{KL}(P \mid \mid Q) = \sum P(x) \log \frac{P(x)}{Q(x)}\)

InfoNCE

1. 什么是 InfoNCE？

InfoNCE (Information Noise-Contrastive Estimation) 是由 Google DeepMind 在 2018 年提出的（出现在著名的 CPC 论文中）。 它的目标是：从一堆“噪声样本”（Negative samples）中，准确地识别出那个“正确样本”（Positive sample）。

直观理解：

想象你在玩一个“找茬”游戏。

  • 锚点 (Anchor)：一张猫的照片。
  • 正样本 (Positive)：同一张猫经过旋转或裁剪后的照片。
  • 负样本 (Negatives)：100 张狗、树、汽车的照片。
  • 目标：让模型在特征空间里，把“锚点”和“正样本”拉得尽可能近，同时把“锚点”和那 100 个“负样本”推得尽可能远。

2. 数学公式

InfoNCE 的公式看起来非常像我们熟悉的 Softmax + 交叉熵： \(\mathcal{L}_{InfoNCE} = -\mathbb{E} \left[ \log \frac{\exp(sim(z_i, z_j) / \tau)}{\exp(sim(z_i, z_j) / \tau) + \sum_{k=1}^{K} \exp(sim(z_i, z_{n,k}) / \tau)} \right]\)

  • \(sim(z_i, z_j)\)：通常是余弦相似度，衡量两个向量有多像。
  • \(z_i\) 和 \(z_j\)：正样本对（Positive Pair）。
  • \(z_{n,k}\)：负样本（Negative Samples）。
  • \(\tau\) (Temperature)：温度参数，用来控制模型对“困难负样本”的关注程度。
  • 分母：是一个正样本相似度 + 所有负样本相似度的总和。

3. 它和交叉熵、KL 散度有什么关系？

为什么叫这个名字？我们可以从两个角度拆解：

  1. 分类视角（Categorical Cross Entropy）： 如果我们把这个任务看成一个 \(K+1\) 类的分类问题（其中只有第 1 类是正确的），那么 InfoNCE 就是在对这个分布计算交叉熵。标签是 One-hot 的 \([1, 0, 0, \dots, 0]\)。
  2. 信息论视角（Mutual Information）： “Info” 代表 Mutual Information (互信息)。 数学上可以证明，最小化 InfoNCE 损失，等价于最大化“锚点”和“正样本”之间互信息的下界。
    • 通过推开负样本，我们实际上是在减少噪声，从而让模型学到的特征包含更多关于输入数据的本质信息。

4. 关键参数：温度系数 \(\tau\) 的作用

在 InfoNCE 中，\(\tau\) 非常关键：

  • \(\tau\) 较小：模型会非常关注那些“离得近”的负样本（困难负样本），损失函数的分布变得很尖锐。
  • \(\tau\) 较大：模型对所有负样本“一视同仁”，分布变得平滑。

5. 应用场景

这是目前自监督学习（Self-Supervised Learning）的基石，著名的模型包括：

  • CV 领域：SimCLR, MoCo, CLIP (图文对齐)。
  • NLP 领域：SimCSE (句向量表征)。
  • 推荐系统：双塔模型中的负采样训练。

6.具体计算

1. 期望 \(\mathbb{E}\) 到底是什么？

在 InfoNCE 的原始公式中： \(\mathcal{L}_{InfoNCE} = -\mathbb{E}_X \left[ \log \frac{f(x, x^+)}{\sum_{j=1}^{K} f(x, x_j)} \right]\) 这里的 期望 \(\mathbb{E}\) 在实际代码实现中，指的就是对 Batch 内所有样本求平均值。

  • 每一个样本（比如“猫”）都会作为一个 Anchor 计算一次损失。
  • 将 Batch 中所有样本产生的损失加起来，除以样本总数。

2. 具体计算流程（以 3 张图片为例）

假设你的 Batch 包含：

  • 样本 A (猫)：增强版本为 \(a_1, a_2\)
  • 样本 B (狗)：增强版本为 \(b_1, b_2\)
  • 样本 C (汽车)：增强版本为 \(c_1, c_2\) 目前我们总共有 6 个特征向量。在对比学习（如 SimCLR）中，通常会计算一个 \(6 \times 6\) 的相似度矩阵。
第一步：计算相似度矩阵 (Similarity Matrix)

利用你提到的 torch.mm，将这 6 个归一化后的向量与其转置相乘： \(Mat = \text{Features} \times \text{Features}^T\) 得到的矩阵中，每个元素 \(s_{i,j}\) 代表第 \(i\) 个增强图和第 \(j\) 个增强图的余弦相似度。

第二步：寻找“正样本对” (Positive Pairs)

在这个 \(6 \times 6\) 的矩阵中：

  • \((a_1, a_2)\) 是正样本对。
  • \((b_1, b_2)\) 是正样本对。
  • \((c_1, c_2)\) 是正样本对。
  • 注意：\((a_1, a_1)\) 是自己跟自己比，通常会被 Mask（掩码）掉，不参与计算。
第三步：针对单个样本计算损失 (以 \(a_1\) 为例)

当 \(a_1\) 作为 Anchor 时：

  • 分子：\(\exp(sim(a_1, a_2) / \tau)\) —— 这是它找到“亲兄弟”的得分。
  • 分母：\(\exp(sim(a_1, a_2) / \tau) + \sum \exp(sim(a_1, \text{others}) / \tau)\) —— 这是它跟 Batch 内除了自己以外所有 5 个样本的得分总和。
    • 负样本包括：\(b_1, b_2, c_1, c_2\)（不同类的图）。
    • 在 SimCLR 等算法中，\(a_1\) 的负样本通常也包括 \(b_1, b_2, c_1, c_2\)。
第四步：求总和与平均 (期望的体现)

你会得到 6 个 Loss 值（每个增强版本都做一次 Anchor）： \(\mathcal{L}_{total} = \frac{1}{6} (\mathcal{L}_{a1} + \mathcal{L}_{a2} + \mathcal{L}_{b1} + \mathcal{L}_{b2} + \mathcal{L}_{c1} + \mathcal{L}_{c2})\) 这个“除以 6”的操作，就是在计算数学公式里的期望。

3. 总结

概念 在你的例子中的具体含义
正样本 (Positive) 来自同一张图的不同增强（如 \(a_1\) 和 \(a_2\)）。
负样本 (Negative) Batch 中所有来自其他原始图片的增强（如 \(a_1\) 的负样本是 \(b, c\) 系列）。
期望 (Expectation) 对这 6 个增强样本分别计算出的 Loss 取算术平均数。

余弦相似度

简单来说，余弦相似度（Cosine Similarity） 衡量的是两个向量在方向上的一致性，而不关心它们的“长度”或“大小”。 在数学上，它是两个向量之间夹角的余弦值。

1. 数学公式

给定两个 \(n\) 维向量 \(A\) 和 \(B\)，其余弦相似度计算如下： \(\text{similarity} = \cos(\theta) = \frac{A \cdot B}{\\mid A\\mid \\mid B\\mid } = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \sqrt{\sum_{i=1}^{n} B_i^2}}\)

  • 分子：向量的点积。
  • 分母：向量模长（长度）的乘积。这个除法操作实际上是对向量进行了“归一化”，消除了长度的影响。

2. 取值范围与含义

余弦相似度的取值范围在 \([-1, 1]\) 之间：

数值 含义 几何关系
1 完全相同 两个向量指向同一个方向（夹角为 0°）。
0 无关/正交 两个向量相互垂直（夹角为 90°）。
-1 完全相反 两个向量指向相反的方向（夹角为 180°）。

注意：在文本处理或推荐系统中，向量分量通常是非负的（如词频），此时余弦相似度的范围通常在 \([0, 1]\) 之间。

3. 为什么它很重要？（与欧氏距离的区别）

这是面试中经常被问到的点。欧氏距离衡量的是空间中两点之间的“绝对距离”，而余弦相似度衡量的是“相对方向”。

  • 例子：假设我们要比较两篇文章的相似性。
    • 文章 A 提到了 10 次“人工智能”。
    • 文章 B 提到了 100 次“人工智能”。
    • 如果用欧氏距离，这两篇文章可能因为长度差异巨大而被判定为“不相似”。
    • 如果用余弦相似度，只要它们谈论的主题比例一致，它们就被认为是“高度相似”的。

4. 常见应用场景

  1. 文本挖掘/NLP：比较文档或单词嵌入（Word Embeddings）的语义接近度。
  2. 推荐系统：计算用户兴趣向量或商品特征向量的相似度（例如：即便一个用户看电影多，一个看电影少，只要口味一致，余弦相似度就很接近）。
  3. 人脸识别：比较提取出的人脸特征向量。

5. 与 InfoNCE 的联系

回到你之前问的 InfoNCE Loss。在计算 InfoNCE 时，公式分子里的 \(sim(z_i, z_j)\) 通常指的就是余弦相似度。 通过将向量投影到超球面上（即做归一化，只看角度），模型可以更专注于学习语义特征的分布，而不会被特征向量的数值大小（模长）所干扰。
